# Render Deployment Guide

## Overview
This guide explains how to deploy the Library Management System to Render.

## Files Created for Render Deployment

### 1. **app.py** - Main Application Entry Point
- Configured for Render environment
- Uses PORT environment variable
- Production-ready settings

### 2. **Procfile** - Process Definition
- Tells Render how to run the application
- Uses Gunicorn for production

### 3. **runtime.txt** - Python Version
- Specifies Python 3.9.16 for Render

### 4. **requirements.txt** - Dependencies
- Includes Gunicorn for production server
- All necessary Flask packages

### 5. **render.yaml** - Render Configuration
- Complete service configuration
- Environment variables set
- Build and start commands

### 6. **setup_database.py** - Database Initialization
- Creates database and tables
- Sets up admin user
- Adds sample data

### 7. **build.sh** - Build Script
- Installs dependencies
- Initializes database

### 8. **.gitignore** - Git Ignore File
- Excludes unnecessary files
- Protects sensitive data

## Deployment Steps

### 1. Push to GitHub
```bash
git add .
git commit -m "Ready for Render deployment"
git push origin main
```

### 2. Deploy to Render
1. Go to [Render Dashboard](https://dashboard.render.com)
2. Click "New +" → "Web Service"
3. Connect your GitHub repository
4. Select the `College-library-management` repository
5. Render will automatically detect the configuration

### 3. Configuration
- Render will use the `render.yaml` file automatically
- Environment variables will be set
- Build process will run automatically

## Troubleshooting

### Common Issues

1. **Build Fails**
   - Check requirements.txt for correct versions
   - Verify Python version in runtime.txt

2. **Database Errors**
   - Ensure setup_database.py runs correctly
   - Check DATABASE_PATH environment variable

3. **Login Issues**
   - Verify admin credentials in environment variables
   - Check database initialization

### Logs
- Check Render dashboard for build logs
- Application logs available in dashboard
- Database initialization logs in build output

## Notes

- Database is stored in `/tmp` (ephemeral storage)
- For production, consider using Render's PostgreSQL
- Free tier has limitations on usage and uptime
- Custom domain can be configured in Render dashboard
